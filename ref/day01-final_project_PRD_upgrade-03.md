## 1. 목표 인지 & 배경

### 1.1 궁극 목표 (One-liner)

> **"글자만 뽑는 생성기에서 → 카피와 실사 이미지가 한 몸으로 살아 숨쉬는, 손에 잡히는 프리미엄 비주얼 카피 스튜디오로."**

v3.0까지 카피의 **신선함·안정성·추적성·기본 디자인(HIG·글래스)** 을 확보했다. 이번 최종 고도화는 카피에 **실사 이미지라는 시각적 실체**를 부여해, 결과물이 "읽는 텍스트"가 아니라 **"바로 쓸 수 있는 비주얼 시안"** 으로 격상되게 한다. 목표는 두 층위다.
1. **기능적:** 생성 카피에 어울리는 Unsplash 실사 이미지를 자동 페어링해 함께 노출(R11).
2. **정서적:** 이미지×글래스의 결합으로 화면 전체에 **생동감과 고급감**을 부여(R12).

### 1.2 이번 고도화가 건드리는 구조적 전환점

이 고도화는 v1.0 이래 유지해 온 **"클라이언트 단독·외부 의존 없음"** 원칙에 **첫 외부 서비스 의존성**을 도입한다. 이는 가볍게 넘길 사안이 아니며, §2에서 **제약 개정**으로 정식 처리한다.

## 2. ⚠ 핵심 제약 충돌 분석 & 개정 (Senior Decision)

### 2.1 충돌의 실체

| 이 프로젝트의 제약 (v1~v3) | Unsplash API 요구 | 충돌 |
|---------------------------|-------------------|------|
| 서버 없음 · 클라이언트 단독 | **Access Key는 기밀 유지 필요.** 클라이언트 접근 시 **프록시 권장/요구** | 키를 클라이언트에 노출하면 가이드라인 위반 + 프로덕션 승인 거부 위험 |
| 외부 API 미전송 (데이터 미유출) | 검색어(keyword)를 Unsplash로 전송해야 이미지 검색 가능 | 사용자 입력 일부가 외부로 나감 |
| 오프라인/독립 동작 | 개발 모드 **시간당 50요청** 제한, 네트워크 필수 | 레이트리밋·오프라인 시 기능 불가 |
| 어떤 자산도 재호스팅 안 함 | **핫링크 필수**(Unsplash CDN URL 직접 사용) + **어트리뷰션 필수** + **다운로드 트리거 필수** | 미준수 시 가이드라인 위반 |

### 2.2 해결안 비교

| 안 | 내용 | 컴플라이언스 | 제약 영향 | 판정 |
|----|------|--------------|-----------|------|
| **A. 서버리스 프록시** | Vercel Functions에 **무상태 프록시 1개**(키 서명·검색 중계 전용, 저장 없음) | ✅ 키 기밀 유지·가이드라인 준수 | "서버 없음"→"무상태 프록시 1개"로 **최소 개정**. DB·데이터 저장은 여전히 없음 | **✅ 채택(권장)** |
| B. 클라이언트에 키 노출 | 번들에 Access Key 임베드 | ❌ 가이드라인 위반, 쿼터 도용·승인 거부 위험 | 제약 유지되나 규정 위반 | ❌ 기각 |
| C. 큐레이션 로컬 이미지 | 카테고리별 무료 이미지 소량 번들 | ✅ (외부 의존 없음) | 제약 100% 유지하나 **"어떤 키워드든 매칭"** 유연성 상실 | 🔸 **폴백으로 병행** |

### 2.3 제약 개정 결정 (D-Amend)

- **개정:** "서버 없음" → **"상태를 저장하지 않는 서버리스 프록시 1개(Unsplash 키 서명·검색 중계 전용)만 허용"**. 여전히 **DB 없음 · 이력/피드백 등 사용자 데이터는 외부 미전송 · 전 데이터 로컬 영속화**는 불변.
- **데이터 최소화:** 프록시로 나가는 것은 **검색 키워드뿐**(이력·리액션·코멘트는 절대 전송 금지). 이미지 기능은 **명시적 opt-in/설정 가능**하며, 꺼지거나 실패하면 v3.0 글래스 표면으로 **무손실 폴백**.
- **회복 탄력성:** 레이트리밋·오프라인·키 부재 시 **C안(큐레이션 로컬 이미지) 또는 글래스 그라데이션 폴백**으로 UI가 깨지지 않게 한다.

> 이 개정을 명시적으로 승인해야 F20 이하가 진행된다. 미승인 시 대안은 **C안 단독(로컬 큐레이션)** 으로 축소 진행.

## 3. 요구사항 분석 — 의도와 목적 해석

### [R11] 카피에 어울리는 실사 이미지 페어링
- **표면 요청:** 생성 카피 옆/위에 어울리는 실사 사진을 같이 보여달라.
- **진짜 의도:** *카피의 "완성된 시안화".* 마케터는 카피 한 줄만으로 설득되지 않는다. 비주얼과 결합된 순간 "실제 광고/썸네일/배너로 이게 어떻게 보일지"가 즉시 상상되고, 채택 판단이 빨라진다. 이미지는 장식이 아니라 **의사결정 촉진 장치**다.
- **해결 문제:** 텍스트-only 결과의 추상성 → "실제로 쓰면 어떤 느낌?"의 공백.
- **성공한 모습:** 각 카피 카드에 입력(키워드·혜택·톤)에서 도출된 **문맥 적합 실사 이미지**가 자동으로 붙고, 어트리뷰션과 함께 노출되며, 이력에서도 동일 이미지가 재현된다(무작위 아님).

### [R12] UI 전체 생동감 + 고급감 (궁극 목표)
- **표면 요청:** UI를 생동감 있고 고급스럽게 다듬어달라.
- **진짜 의도:** *"지금 잘 만든, 사고 싶은 프로덕트"의 인상.* 생동감(Vibrancy)은 실사 이미지·미묘한 모션·색의 생기에서, 고급감(Premium)은 절제된 여백·유리 질감의 깊이·타이포 위계에서 나온다. v3.0의 글래스(F19)는 "표면 마감"이었고, 이제 그 유리 아래에 **실사라는 생명**을 넣어 완성한다.
- **해결 문제:** 이미지 없는 글래스는 자칫 공허(빈 유리)해 보일 수 있음 → 깊이의 대상 부재.
- **성공한 모습:** 실사 사진 위에 유리 카피 패널이 얹혀 **깊이·생기·프리미엄**이 동시에 살아나고, 가독성·접근성·성능은 그대로 지켜진다.

### 3.1 설계 결정 (Design Decisions)

| # | 결정 | 근거 |
|---|------|------|
| D6 | 이미지는 **엔트리에 캐싱 저장**(url·작가·download_location 등) | R11의 "이력 재현" + 불필요한 재페치/쿼터 절약. v2~v3의 재현성 원칙 계승 |
| D7 | 검색 쿼리는 **구조화 입력(keyword 우선 + benefit/톤 무드)** 에서 도출 | 헤드라인 문장 전체보다 입력 키워드가 이미지 검색에 더 정확·저비용 |
| D8 | 한국어 키워드 → **경량 카테고리 사전 + 원문 폴백** | LLM 없이 클라이언트에서 번역 불가. 마케팅 상용어 소사전으로 영문 쿼리 품질 확보, 없으면 원문 질의 |
| D9 | **이미지 위 텍스트는 스크림(대비 오버레이) 필수** | F18의 AA 대비 하한이 F22에 우선. 실사 위 가독성 보장 |
| D10 | 이미지는 **opt-in·비동기·폴백 우선** | 외부 의존을 핵심 경험의 필수 전제로 만들지 않음(핵심은 카피, 이미지는 증폭) |

## 5. 신규 · 변경 기능 명세 (F20~F22)

### F20. Unsplash 실사 이미지 연동 기반 — [Must] (R11)

- **개요:** Unsplash 검색·핫링크·컴플라이언스를 처리하는 연동 계층과 **무상태 서버리스 프록시**를 구축한다.
- **동작:**
  - **프록시(§2.3):** Vercel Function이 Access Key를 서버에서 보관하고 검색·다운로드 트리거 요청을 서명·중계. 저장·로깅 없음, 검색어만 통과.
  - **검색:** `/search/photos`에 쿼리·orientation·`content_filter=high`로 후보 수 개 요청.
  - **핫링크:** 반환된 `urls.*`(CDN)를 **직접 임베드**(재호스팅 금지). 반응형 파라미터(`w/q/fm/dpr`)로 사이즈 최적화.
  - **어트리뷰션:** "Photo by {작가} on Unsplash" + 작가/Unsplash 링크(UTM 파라미터 포함) 표시.
  - **다운로드 트리거:** 사용자가 이미지를 **채택/복사/즐겨찾기 등 "사용" 행위** 시 `download_location`을 비동기 GET(ixid 포함).
  - **레이트리밋/오프라인:** 실패·429·오프라인 시 폴백(로컬 큐레이션 또는 글래스 그라데이션), 사용자 안내.
- **Done when:** 검색→표시→어트리뷰션→다운로드 트리거 4종 정상, 키가 클라이언트에 노출되지 않음, 실패 시 폴백 100%.

### F21. 카피-이미지 페어링 — [Must] (R11)

- **개요:** 생성 입력에서 이미지 검색 쿼리를 도출하고, 후보 중 하나를 선택해 **엔트리에 저장(재현)** 한다.
- **동작:**
  - **쿼리 도출(D7·D8):** keyword 우선, benefit/톤으로 무드 보강. 한국어는 경량 카테고리 사전으로 영문화, 미스 시 원문 질의.
  - **선택:** 후보 중 orientation·관련도 기준 1장 채택. 카드 간 동일 이미지 중복 최소화.
  - **저장/재현:** 선택 이미지를 GenerationEntry에 캐싱(§6). 이력·재방문 시 동일 이미지 표시. **재생성(🔄) 시 새 이미지 재페치.**
  - **품질 우선:** 적합 이미지가 없으면 억지 매칭 대신 폴백(빈 유리 표면 유지).
- **Done when:** 페어링 성공률 ≥90%, 이력 재현 시 동일 이미지, 재생성 시 갱신, 실패 시 폴백.

### F22. 프리미엄 비주얼 컴포지션 — [Must] (R12)

- **개요:** 실사 이미지와 글래스(F19)를 결합해 생동감·고급감을 완성하되 접근성·성능을 지킨다.
- **동작:**
  - **컴포지션:** 카드 배경에 실사 이미지 → 그 위에 **유리 카피 패널**(글래스) → 그 위 카피 텍스트. iOS 스타일 깊이감.
  - **대비 스크림(D9):** 이미지·텍스트 사이 그라데이션/블러 스크림으로 **AA 대비 보장**.
  - **로딩:** `blur_hash` 기반 **blur-up** + 스켈레톤으로 부드러운 등장(생동감).
  - **반응형·성능:** 지연 로딩, 뷰포트별 이미지 사이즈, 동시 페치 상한.
  - **모션 배려:** `prefers-reduced-motion` 시 페이드/블러업 최소화. `prefers-reduced-transparency` 시 스크림 강화(불투명).
  - **폴백 미감:** 이미지 없을 때도 v3.0 글래스 그라데이션으로 **여전히 고급**.
- **Done when:** 이미지×글래스 카드가 AA 대비 유지, blur-up 로딩 자연스러움, 저사양·reduced-motion·오프라인에서 미감·성능 유지.

## 6. 데이터 모델 변경 (GenerationEntry v4)

v3 스키마에 **이미지 필드만 추가**(하위호환 100%, 전부 nullable·기본값).

```javascript
{
  // ── v1~v3 기존 필드 (변경 없음) ──
  id, headline, formula, qualityScore, inputs, variationSeed,
  isFavorite, reaction, comment, generatedAt,
  reactedAt, commentUpdatedAt, actor, displayOrder,

  // ── v4 신규: 실사 이미지 (F20/F21) ──
  image: {
    id,                    // Unsplash 사진 ID
    urls: { thumb, small, regular },  // 핫링크 CDN URL (재호스팅 금지)
    blurHash,              // blur-up 로딩용
    altText,              // 접근성 alt (사진 설명)
    query,                // 실제 사용된 검색 쿼리 (재현·디버깅)
    photographer: { name, profileUrl },  // 어트리뷰션 (필수)
    downloadLocation,     // 다운로드 트리거 엔드포인트 (필수)
    downloadTriggered: false
  } | null,               // 없음/폴백 시 null
  imageStatus: "none"     // none | loading | ready | failed (UI 상태)
}
```

- **마이그레이션:** 신규 필드 nullable → 기존 이력은 `image: null / imageStatus: "none"` 으로 읽어 폴백 표면 렌더. 별도 변환 불필요.
- **저장소:** 기존 IndexedDB 그대로. 이미지 **원본을 저장하지 않고 URL·메타만 캐싱**(용량·핫링크 원칙 준수).
- **재현성:** 엔트리에 이미지가 고정되어 이력/재방문 시 동일. 재생성 시에만 갱신.

## 7. 단계별 업무 전략 (Execution Strategy)

접근성·컴플라이언스 회귀를 막기 위해 **기반 → 페어링 → 비주얼 → 하드닝** 순서로 진행한다.

| Phase | 산출 | 핵심 체크 |
|-------|------|-----------|
| **P0. 제약 개정 승인** | §2.3 개정안 확정 (프록시 vs 큐레이션 단독) | 키 기밀·데이터 최소화 원칙 합의 |
| **P1. F20 연동 기반** | 서버리스 프록시 · 검색 · 핫링크 · 어트리뷰션 · 다운로드 트리거 · 레이트리밋/폴백 | 컴플라이언스 4종, 키 미노출, 폴백 |
| **P2. F21 페어링** | 쿼리 도출(한↔영 사전) · 후보 선택 · 엔트리 캐싱 · 재생성 갱신 | 성공률·재현성 |
| **P3. F22 프리미엄 비주얼** | photo×글래스 컴포지션 · 대비 스크림 · blur-up · 반응형 · 모션 배려 | AA 대비·생동감·성능 |
| **P4. 하드닝** | 성능(지연로딩·사이즈)·접근성(alt·reduced-*)·오프라인·쿼터 캐시 | KPI 전수 검증 |

권장: **P0 승인 → P1 → P2 → P3 → P4.** P1에서 컴플라이언스를 먼저 못 박아야 이후 비주얼 작업이 규정 위반 없이 얹힌다.

## 8. 예외 · 엣지 케이스 & 제약 (계승 + 신규)

- **레이트리밋(429)·오프라인·키 부재:** 로컬 큐레이션 또는 글래스 폴백, 사용자 안내. UI 무손실.
- **적합 이미지 없음:** 억지 매칭 금지 → 폴백 표면 유지.
- **한국어 검색 약세:** 카테고리 사전으로 영문화, 미스 시 원문 질의(그래도 실패면 폴백).
- **부적절 이미지 노출:** `content_filter=high` 적용, 그래도 보장 불가 → 재생성/신고 경로.
- **이미지 위 가독성:** D9 스크림 필수, 미달 위험 시 스크림 강화.
- **컴플라이언스:** 어트리뷰션·다운로드 트리거·핫링크·키 기밀 미준수 배포 금지.
- **유지 제약(개정 후 불변):** DB 없음 · 사용자 이력/피드백 외부 미전송 · 로컬 영속화 · v1~v3 데이터 무손실 하위호환. (서버는 "무상태 프록시 1개"로 한정 개정)

## 10. 성공 기준

- **기능적:** §4 KPI 전 항목 충족(페어링 ≥90%, 컴플라이언스 4/4, 폴백 100%).
- **정서적(궁극 목표):** "생동감 있다 / 고급스럽다 / 바로 쓰고 싶다"는 반응. 스크린샷 기반 디자인 리뷰(web-design-reviewer)로 검수.
- **원칙 보존:** 사용자 데이터 외부 미전송·로컬 영속화·하위호환 불변, 접근성(AA) 유지.

## 11. 향후 백로그 (Post-v4.0)

- 이미지 오리엔테이션/컬러 톤을 톤 프리셋과 연동(무드 페어링 고도화).
- 사용자가 후보 중 이미지 직접 교체(수동 리롤).
- 즐겨찾기 카피+이미지를 **완성 시안 카드(다운로드/공유)** 로 내보내기.
- 로컬 큐레이션 셋 확대로 오프라인 경험 강화.

## 부록. 용어 (신규분)

- **핫링크(Hotlinking):** 이미지를 재호스팅하지 않고 Unsplash CDN URL을 직접 임베드하는 필수 방식.
- **다운로드 트리거:** 사용자가 사진을 "사용"할 때 `download_location`을 호출해 다운로드 수를 집계하는 필수 이벤트.
- **어트리뷰션:** 사진작가·Unsplash 명시 및 프로필 링크(UTM 포함) 표시 의무.
- **무상태 프록시:** 키 서명·검색 중계만 하고 어떤 데이터도 저장하지 않는 서버리스 함수.
- **대비 스크림(Scrim):** 이미지와 텍스트 사이에 두어 가독성(AA)을 보장하는 반투명 오버레이.
- **blur-up:** 저해상도(blur_hash)에서 원본으로 부드럽게 전환하는 이미지 로딩 기법.
