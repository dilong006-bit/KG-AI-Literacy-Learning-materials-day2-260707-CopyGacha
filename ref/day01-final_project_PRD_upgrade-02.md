## 1. 고도화 배경 — v2.0의 잔여 마찰(Residual Friction)

v2.0은 "축적·학습하는 카피 워크스페이스"를 완성했으나, **실사용 밀도가 높아질수록 드러나는 4가지 마찰**이 남아 있다. 이번 고도화는 새 기능 확장이 아니라, **이미 좋은 제품의 신뢰도·완성도(polish)를 끌어올리는** 데 초점을 둔다.

1. **생성 다양성의 한계 (R6):** 엔진이 직전 문장 1건과만 중복 비교하고, 6개 공식이 서로를 모른 채 독립 생성된다. 공식당 템플릿 5개 + 숫자풀 5개로 조합이 빠르게 소진돼, 재생성·재검색 시 **같은 카피가 반복 노출**된다. → "가챠"의 핵심 가치인 *뽑는 재미와 신선함*이 훼손된다.
2. **정렬의 불안정성 (R7):** 인기 학습 점수(StyleScore)가 **엔트리 변경마다 즉시 재계산**되고, 결과 그리드가 그 랭킹으로 **매 렌더 실시간 재정렬**된다. 그 결과 리액션 한 번에 눈앞의 카드가 위아래로 튀어 **읽기·비교의 맥락이 끊긴다.**
3. **피드백의 추적 불가 (R8):** 코멘트·리액션이 **타임스탬프 없이** 최종 값만 덮어쓰기로 저장된다. "이 메모 언제 남겼더라?"를 되짚을 수 없어, 피드백이 **일회성 낙서**에 머문다.
4. **시각 완성도의 시대 격차 (R9·R10):** 일반적 인디고+회색 토큰, 불투명 카드, 표준 그림자로 구성돼 **최신 프리미엄 UI의 체감(HIG의 명료성·깊이, 글래스 머티리얼)** 에 미치지 못한다. 터치 타깃·대비·다크모드 등 **접근성 기반**도 정리되어 있지 않다.

### 고도화 방향성 (One-liner)

> **"기능이 갖춰진 워크스페이스에서 → 신뢰할 수 있고, 매 순간 신선하며, 손에 닿는 완성도의 프로덕트로."**

단, v1.0/v2.0의 핵심 제약(**서버·DB 없음, 클라이언트 단독, 외부 API 미전송, 전 데이터 로컬 영속화, 기존 이력 무손실 하위호환**)은 **전부 그대로 유지**한다. 본 고도화는 이 제약 안에서 100% 달성 가능하다.

## 2. 요구사항 분석 — 의도와 목적 해석

각 요구사항을 "표면 요청 → 진짜 의도 → 해결 문제 → 성공한 모습" 순으로 해석한다.

### 2.1 요구사항별 심층 분석

**[R6] 동일한 카피가 다수 노출됨**
- **표면 요청:** 중복 카피가 뜨지 않게 해달라.
- **진짜 의도:** *생성의 신선함과 신뢰.* "가챠"는 매 회차 **새로운 뽑기**라는 기대가 핵심이다. 같은 문장이 반복되면 엔진이 "고장 났거나 얕다"는 인상을 주고, 6개 시안을 나란히 비교하는 실무 흐름도 무의미해진다.
- **해결 문제:** 좁은 조합 공간 + 근시안적 중복 검사(직전 1건) → 반복.
- **성공한 모습:** ① 한 세트(6개) 안에서 완전 동일 카피 0건. ② 같은 입력으로 반복 생성/재생성해도 최근 노출분과 겹치지 않는다. ③ 조합이 소진되면 반복 대신 "표현 변주(어휘·어순·수식)"로 다양성을 만들어낸다.

**[R7] 리액션 즉시 정렬 반영 → 산만함**
- **표면 요청:** 리액션할 때 카드가 재정렬되지 않게 해달라.
- **진짜 의도:** *조작 중 시각적 안정성(spatial stability).* 사용자는 "지금 보고 있는 6개"를 **고정된 좌표**에서 비교·평가하고 싶다. 평가 행위(리액션)가 평가 대상(카드 위치)을 흔드는 것은 인터랙션 설계의 자기모순이다.
- **해결 문제:** 인기 학습(F12)의 실시간 랭킹이 *현재 결과 뷰*에까지 즉시 침투 → 카드 점프.
- **성공한 모습:** 리액션을 눌러도 **현재 세트의 카드 순서는 불변**. 인기 학습은 그대로 동작하되, 그 결과는 **다음 생성부터** 반영된다(현재 뷰는 생성 시점 순서를 유지).

**[R8] 누가 언제 남긴 코멘트인지 알기 어려움**
- **표면 요청:** 코멘트에 작성자·시각을 표시해달라. (단, 현재 로그인 미도입 → "누가"는 스킵 가능)
- **진짜 의도:** *피드백의 맥락 보존과 신뢰.* 메모가 "언제 시점의 판단인지" 알아야, 나중에 되짚어 재현·수정 결정을 내릴 수 있다. 로그인은 없지만, 향후 협업/로그인 도입 시 **작성자 귀속을 붙일 수 있는 구조**를 지금 확보해두는 것이 상위 의도다.
- **해결 문제:** 타임스탬프 부재 + 덮어쓰기 저장 → 맥락 소실.
- **성공한 모습:** 코멘트·리액션에 **작성/수정 시각이 상대시간("3분 전")으로 표시**된다. 데이터 스키마는 **작성자(actor) 필드를 nullable로 미리 확보**해, 추후 로그인만 붙이면 "누가"가 무손실로 채워진다.

**[R9] HIG(휴먼 인터페이스 가이드라인) 준수**
- **표면 요청:** Apple HIG를 따르는 UI로 다듬어달라.
- **진짜 의도:** *명료성·일관성·접근성이라는 프로덕트 기본기.* HIG는 장식이 아니라 **명료성(Clarity)·순응(Deference)·깊이(Depth)** 원칙과 **최소 44pt 터치 타깃, 명확한 상태, 대비, 다크모드, 모션 배려** 같은 검증된 기준의 총합이다. "예쁘게"가 아니라 "누구에게나 잘 작동하게"가 목적이다.
- **해결 문제:** 임의 토큰·작은 터치 타깃·다크모드/포커스 상태 미정의 → 접근성·일관성 공백.
- **성공한 모습:** ① 모든 인터랙티브 요소 ≥ 44×44pt. ② 본문 대비 ≥ WCAG AA(4.5:1). ③ 라이트/다크모드 완전 대응. ④ 포커스 링·눌림 상태 일관. ⑤ `prefers-reduced-motion` 존중. ⑥ HIG의 타이포 스케일·간격 그리드에 정렬된 디자인 토큰 세트.

**[R10] UI 트렌드 반영 — 글래스모피즘**
- **표면 요청:** 글래스모피즘(유리 질감)을 입혀달라.
- **진짜 의도:** *동시대적 프리미엄 체감.* 반투명·블러·깊이 레이어로 "지금 잘 만든 앱"의 인상을 준다. 단 트렌드를 **접근성과 성능을 깨지 않는 선**에서 적용하려는 것이 진짜 의도다(유리 위 텍스트 가독성 유지).
- **해결 문제:** 불투명 카드·평면 그림자 → 시대 격차, 깊이감 부재.
- **성공한 모습:** 카드·헤더·팝오버 등 표면에 **backdrop blur + 반투명 배경 + 상단 하이라이트 보더 + 레이어드 배경**의 글래스 머티리얼이 적용되되, **텍스트 대비 AA 유지 · 미지원/절전 환경 솔리드 폴백 · 투명도 축소 접근성 설정 존중**이 함께 보장된다.

### 2.2 요구사항 상호관계 (설계 구조도)

```
[신선함/신뢰 축]                         [완성도/신뢰 축]
 F15 카피 다양성 ──┐                      ┌── F18 HIG 기반(토큰·접근성) ──┐
                   ├─▶ 매 순간 새롭고        │                              ├─▶ 손에 닿는
 F16 정렬 안정화 ──┘   흔들리지 않는 결과      └── F19 글래스 머티리얼 ───────┘   프리미엄 체감
                                             (F19는 F18 위에 얹힘: 접근성이 트렌드보다 우선)
 F17 피드백 추적성 ──▶ 피드백이 맥락과 함께 남아 재현·판단의 근거가 된다
```

- **F18 → F19 의존:** 글래스(F19)는 반드시 HIG 접근성 기반(F18) **위에** 얹는다. 유리 효과가 대비·가독성을 깨면 F18이 F19를 override 한다(투명도 축소 폴백).
- **F16 ↔ F12(기존):** 정렬 안정화는 인기 학습(F12)을 **끄는 게 아니라**, 학습 결과의 *적용 시점*을 "현재 뷰 실시간"에서 "다음 생성"으로 옮기는 것이다. 학습 신호 수집은 그대로다.
- **F15 ↔ 이력(F10):** 다양성 판정을 위해 "최근 노출 카피" 참조가 필요하며, 이는 기존 이력(F10)을 재활용한다(신규 저장소 불필요).

### 2.3 핵심 아키텍처 결정 사항 (Design Decisions)

| # | 결정 | 근거 |
|---|------|------|
| D1 | 정렬 순서를 **생성 시점에 1회 확정(snapshot)** 하고 현재 뷰는 그 순서로 고정 | R7의 "산만함"은 *실시간 재정렬*이 원인. 스냅샷이면 리액션이 위치를 흔들지 않으면서 인기 학습은 유지 |
| D2 | 코멘트를 **문자열 유지 + 타임스탬프 필드 추가**(배열/스레드로 확장하지 않음) | 로그인·다중작성자 없는 단일 사용자 로컬 앱. "언제"만으로 R8 의도 충족, 마이그레이션 최소·하위호환 100% |
| D3 | `actor` 필드를 **nullable로 선확보** | 향후 로그인 도입 시 "누가"를 무손실 확장(전방호환) |
| D4 | 다양성은 **재생성 확대(re-roll) + 표현 변주** 2단계로 | 조합 공간이 유한하므로, 소진 시 "실패" 대신 어휘·어순·수식 변주로 신선함 확보 |
| D5 | 디자인 토큰 **전면 재정의(HIG 정렬) 후 글래스는 표면 레이어에만 적용** | 토큰(색·간격·타이포·모션)은 구조, 글래스는 표면 마감. 순서를 지켜야 접근성이 보장됨 |

## 3. 고도화 목표 & 신규 KPI

### 3.1 정성 목표
- 매 생성이 **신선**하다(반복 없음).
- 평가 중 화면이 **흔들리지 않는다**(공간 안정성).
- 피드백이 **맥락과 함께 남는다**(언제).
- 누구에게나 **잘 작동하고**(HIG·접근성), **동시대적으로 세련**되다(글래스).

### 3.2 신규 KPI (R6~R10 직접 대응)

| KPI | 정의 | 목표 |
|-----|------|------|
| 세트 내 중복률 | 한 생성 세트(6개) 중 완전 동일 카피 비율 | **0%** |
| 재생성 반복률 | 동일 입력 연속 재생성 시 직전 대비 동일 카피 비율 | **≤ 5%** (직전 노출과 겹치지 않음) |
| 정렬 안정성 | 리액션·코멘트 후 현재 세트 카드 위치 변동 수 | **0** |
| 피드백 타임스탬프 커버리지 | 신규 코멘트·리액션 중 시각 기록 보유 비율 | **100%** |
| 접근성 통과율 | 터치타깃(≥44pt)·대비(≥AA)·다크모드·reduced-motion 4항목 통과 | **100%** |
| 글래스 폴백 안전성 | backdrop-filter 미지원/투명도 축소 시 가독성 유지 | **100%** (솔리드 폴백) |

## 4. 신규 · 변경 기능 명세

각 기능은 ID · 우선순위(MoSCoW) · 완료 조건(Done when)을 갖는다.

### F15. 카피 다양성 엔진 (Engine v3) — [Must] (R6)

- **개요:** 생성 엔진에 **전역 중복 회피 + 표현 변주** 계층을 추가한다.
- **동작:**
  - ① **세트 내 유일성:** 한 세트(6개) 생성 시 이미 채택된 헤드라인 집합과 대조해 완전 동일 문장을 배제한다(공식 간 교차 중복 포함).
  - ② **최근 노출 회피:** 최근 N개(예: 직전 2~3세트) 노출 카피를 이력(F10)에서 조회해 회피 대상에 포함한다.
  - ③ **표현 변주(fallback):** 유한 조합이 소진돼 회피가 어려우면 실패 대신 어휘 치환(파워워드 사전)·어순·수식어·숫자풀 변경으로 변주해 신선도를 확보한다.
  - ④ **품질 우선순위 유지:** 변주는 기존 품질 스코어링(F14) 임계·클리셰 블랙리스트를 계속 통과해야 한다(다양성이 품질을 훼손하지 않음).
- **Done when:** 세트 내 중복 0%, 동일 입력 연속 재생성 시 직전 세트와 중복 ≤5%, 변주 결과도 품질 임계 통과.

### F16. 결과 정렬 안정화 (Order Snapshot) — [Must] (R7)

- **개요:** 현재 결과 뷰의 카드 순서를 **생성 시점에 1회 확정**하고, 이후 인터랙션으로 재정렬하지 않는다.
- **동작:**
  - 생성 시 인기 학습 랭킹(또는 품질순)을 반영해 **표시 순서를 스냅샷**으로 고정한다.
  - 리액션·코멘트·즐겨찾기·복사는 **해당 카드 자리에서 인플레이스 갱신**만 하고 순서에 영향을 주지 않는다.
  - 인기 학습(F12)은 백그라운드로 계속 신호를 축적하되, 그 결과는 **다음 생성 세트부터** 순서에 반영된다.
  - (선택) 사용자가 원할 때 "지금 순서 새로고침" 명시적 액션으로만 재정렬을 허용한다.
- **Done when:** 리액션/코멘트 후 현재 세트 카드 위치 변동 0, 인기 학습 신호 수집·다음 세트 반영은 정상 동작.

### F17. 피드백 추적성 (타임스탬프) — [Must] (R8)

- **개요:** 코멘트·리액션에 작성/수정 시각을 기록하고 상대시간으로 표시한다. 작성자 귀속은 전방호환 필드로 선확보(로그인 도입 전까지 null).
- **동작:**
  - 코멘트 저장 시 `commentUpdatedAt` 기록, 카드에 "3분 전"과 같은 상대시간 표시(정확 시각은 hover/long-press 툴팁).
  - 리액션 설정 시 `reactedAt` 기록.
  - `actor` 필드를 nullable로 스키마에 추가(현재 항상 null → 표시 생략).
  - 기존 항목(타임스탬프 없음)은 `generatedAt`로 폴백하거나 "시간 정보 없음" 표기.
- **Done when:** 신규 코멘트·리액션 100%에 시각 기록·표시, 기존 이력 무손실, `actor` 확장 지점 확보.

### F18. HIG 준수 디자인 시스템 — [Must] (R9)

- **개요:** 디자인 토큰과 컴포넌트를 Apple HIG 원칙(명료성·순응·깊이)과 접근성 기준에 맞춰 재정의한다.
- **동작:**
  - **터치 타깃:** 모든 인터랙티브 요소 최소 44×44pt 히트 영역 보장(현재 작은 리액션/`?` 버튼 포함).
  - **타이포·간격:** HIG 지향 타입 스케일과 8pt 그리드 간격 토큰 정비, 시스템 폰트 스택 유지.
  - **시맨틱 컬러 + 다크모드:** 의미 기반 색 토큰과 라이트/다크 완전 대응.
  - **상태·모션:** 포커스 링·hover·active·disabled 상태 일관 정의, `prefers-reduced-motion` 시 룰렛/트랜지션 축소.
  - **대비:** 본문·주요 텍스트 대비 ≥ AA(4.5:1).
- **Done when:** 44pt·AA대비·다크모드·reduced-motion 4항목 100% 통과, 토큰 세트 문서화.

### F19. 글래스모피즘 머티리얼 — [Should] (R10)

- **개요:** 표면(카드·헤더·탭바·팝오버)에 유리 질감 머티리얼을 적용하되 접근성·성능 가드레일을 함께 둔다.
- **동작:**
  - **머티리얼:** `backdrop-filter: blur()+saturate()` + 반투명 배경 + 상단 하이라이트 보더 + 미세 그림자로 깊이 표현, 배경에 레이어드 그라데이션/블롭으로 유리감 강화.
  - **가드레일(F18 우선):** 유리 위 텍스트 대비 AA 유지, `backdrop-filter` 미지원 시 **솔리드 폴백**, `prefers-reduced-transparency`(투명도 축소) 시 불투명 표면으로 대체, 블러 레이어 수 제한으로 성능 보호.
- **Done when:** 지정 표면에 글래스 적용 + AA 대비 유지 + 폴백/투명도축소 대응 확인, 저사양에서 프레임 저하 없음.

### 우선순위 요약 (MoSCoW)

| 기능 | 우선순위 | 사유 |
|------|----------|------|
| F15 카피 다양성 | **Must** | 제품 핵심 가치(가챠 신선함) 직결 |
| F16 정렬 안정화 | **Must** | 인터랙션 자기모순 해소, 저비용 고효과 |
| F17 피드백 추적성 | **Must** | 저비용, 스키마 전방호환 확보 |
| F18 HIG 기반 | **Must** | 접근성·일관성 기본기, F19의 전제 |
| F19 글래스 머티리얼 | **Should** | 체감 가치 크나 F18 위에 얹혀야 안전 |

## 5. 데이터 모델 변경 (GenerationEntry v3)

기존 스키마(v2)에 **필드 추가만** 하며 기존 필드·타입은 보존한다(하위호환 100%).

```javascript
{
  // ── v2 기존 필드 (변경 없음) ──
  id, headline, formula, qualityScore,
  inputs: { keyword, target, benefit, tone },
  variationSeed,
  isFavorite,
  reaction,               // like | love | fire | hmm | null
  comment,                // 문자열 유지 (D2)
  generatedAt,

  // ── v3 신규 필드 ──
  reactedAt: null,        // F17: 리액션 설정/변경 시각 (ISO), 없으면 null
  commentUpdatedAt: null, // F17: 코멘트 작성/수정 시각 (ISO), 없으면 null
  actor: null,            // F17/D3: 작성자 귀속(전방호환). 로그인 전까지 항상 null
  displayOrder: null      // F16: 세트 내 표시 순서 스냅샷(정수). 없으면 생성순 폴백
}
```

- **마이그레이션:** 신규 필드는 전부 nullable·기본값 존재 → 기존 이력은 **읽는 시점에 기본값으로 채움**(별도 일괄 변환 불필요). 기존 항목의 시각 표시는 `generatedAt` 폴백.
- **저장소:** 기존 IndexedDB(폴백 LocalStorage) 그대로. 신규 저장소·스키마 버전업 없이 필드 확장으로 처리.
- **정렬 스냅샷:** `displayOrder`는 생성 세트 내에서만 유효한 상대 순서. 인기 학습(F12)은 이 값을 *다음 세트* 생성 시에만 산정에 반영.

## 6. 화면 · 정보구조(IA) 변경

- **IA 골격(생성/이력/지표 3탭) 자체는 불변** — 본 고도화는 표면·상호작용 품질 개선이 중심이다.

## 7. 디자인 시스템 상세 (F18 · F19)

> 상세 구현 값(구체 토큰·CSS·컴포넌트 계약)은 **기술명세서(technical-spec)** 로 이관한다. 본 절은 PRD 수준의 원칙·기준·가드레일만 정의한다.

### 7.1 HIG 원칙 매핑 (F18)
- **Clarity(명료성):** 콘텐츠 우선, 장식 최소. 카피 텍스트가 항상 최상위 위계.
- **Deference(순응):** 유리·블러는 콘텐츠를 돕는 배경 레이어로만. 표면이 내용을 압도하지 않음.
- **Depth(깊이):** 레이어·그림자·블러로 계층을 표현하되 과하지 않게.
- **접근성 하한선:** 터치 ≥44pt · 대비 ≥AA · 다크모드 · reduced-motion · 명확한 포커스.

### 7.2 글래스 머티리얼 스펙 (F19)
- **구성:** 반투명 배경 + `backdrop-filter`(blur+saturate) + 상단 1px 하이라이트 보더 + 부드러운 그림자.
- **레이어링:** 페이지 배경에 은은한 그라데이션/블롭 → 유리 표면이 그 위에서 굴절되는 느낌.
- **강도 절제:** 블러 반경·투명도는 텍스트 대비 AA를 깨지 않는 범위로 상한.

### 7.3 접근성 · 성능 가드레일 (F18이 F19에 우선)
- `backdrop-filter` 미지원 브라우저 → **솔리드 표면 폴백**.
- `prefers-reduced-transparency` → 불투명 표면.
- `prefers-reduced-motion` → 룰렛/트랜지션 최소화.
- 블러 레이어 수 제한 + 큰 영역 블러 지양으로 저사양 성능 보호.

## 부록. 용어 (신규분)

- **Order Snapshot:** 생성 세트 내 카드 표시 순서를 생성 시점에 확정해 고정하는 값(`displayOrder`).
- **표현 변주(Variation):** 조합 소진 시 어휘·어순·수식·숫자를 바꿔 신선도를 확보하는 다양성 기법.

- **글래스 머티리얼:** backdrop blur·반투명·하이라이트 보더로 구성한 유리 질감 표면.

## 다음 단계 (Handoff)

본 PRD의 **F15~F19 및 GenerationEntry v3 스키마**를 입력으로 →
1. **technical-spec**: 각 F-ID를 아키텍처·구체 토큰·컴포넌트 계약·엣지케이스 처리로 상세화.
2. **prompt-builder**: 기술명세를 Claude Code용 구현 프롬프트로 변환(F-ID 단위 작업 지시).

F-ID를 매개로 챗(기획)↔코드(구현)의 추적성을 끝까지 유지한다.
