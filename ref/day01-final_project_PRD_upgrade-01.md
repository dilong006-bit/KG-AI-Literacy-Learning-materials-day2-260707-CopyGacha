## 1. 고도화 배경 — v1.0의 구조적 한계

v1.0은 "키워드 → 즉시 6개 시안"이라는 **속도**에 최적화된 반면, 다음의 구조적 한계를 가진다.

1. **휘발성 (Stateless):** 즐겨찾기를 제외한 모든 생성 결과는 새로고침 시 소실된다. "아까 그 좋았던 시안"을 되찾을 수 없다.
2. **맥락 소실:** 즐겨찾기에 입력값이 저장되긴 하나, **어떤 조합(키워드×타깃×혜택×톤×공식)이 좋은 결과를 냈는지** 사용자가 되짚거나 재현할 수 있는 흐름이 없다.
3. **피드백 신호 부재:** 별표(즐겨찾기)라는 이진(binary) 신호뿐. "왜 좋았는지", "어디를 고치고 싶은지"를 남길 방법이 없다.
4. **개인화 부재:** 매 세션이 백지에서 시작. 사용자가 선호하는 스타일을 시스템이 학습하지 못한다.
5. **엔진 품질 편차:** 생성 로직이 단순 템플릿 치환 + 랜덤 백업이라, 결과 품질이 입력에 따라 크게 흔들리고 클리셰가 반복된다.

### 고도화 방향성 (One-liner)

> **"일회성 생성기(Generator)"에서 → "축적되고 학습하는 카피 워크스페이스(Workspace)"로."**

단, v1.0의 핵심 제약(**서버·DB 없음, 클라이언트 단독, 외부 API 미사용**)은 **그대로 유지**한다.

## 2. 요구사항 분석 — 의도와 목적 해석

각 요구사항을 "표면 요청 → 진짜 의도 → 해결 문제 → 성공한 모습" 순으로 해석했다.

### 2.1 요구사항별 심층 분석

**[R1 + R4] 입력 설정값 기반 이력 관리 / 카피 생성 이력 관리**
- **표면:** 생성한 카피를 기록으로 남기고, 어떤 입력으로 만들었는지 붙여둔다.
- **진짜 의도:** *재현성(reproducibility)과 추적성(traceability)*. 좋은 카피는 "우연히 나온 한 줄"이 아니라 **특정 입력 조합의 산출물**이다. 그 조합을 되살려 **재생성·미세조정·비교**할 수 있어야 실무에서 쓸 수 있다.
- **해결 문제:** "그때 그 좋은 카피, 무슨 키워드·타깃으로 뽑았더라?" → 맥락 소실.
- **R1과 R4의 관계:** R1은 **각 이력 항목이 담아야 할 메타데이터(스키마)** 요구이고, R4는 **그 이력들을 조회·검색·필터·삭제·재실행하는 관리 UX** 요구다. → **하나의 "생성 이력" 도메인**으로 통합 설계한다.
- **성공한 모습:** 모든 생성 결과가 입력 맥락과 함께 타임라인에 쌓이고, 한 번의 클릭으로 "동일 조건 재생성" 또는 "입력 불러와 편집"이 가능하다.

**[R3] 리액션 & 코멘트**
- **표면:** 카피 카드에 이모지 리액션과 코멘트를 단다.
- **진짜 의도:** *이진 즐겨찾기를 넘어선 정성·정량 피드백 레이어*. 리액션(👍/❤️/🔥/🤔)은 **가볍고 빠른 감정 신호**로 인기 학습(R2)의 원천 데이터가 되고, 코멘트는 **"톤을 더 세게", "숫자 넣자" 같은 수정 메모**로 다음 생성의 방향을 잡아준다.
- **해결 문제:** 피드백 신호 부재 → 시스템이 사용자의 취향을 알 수 없음.
- **성공한 모습:** 사용자가 결과에 반응을 남기고, 그 반응이 개인화·추천의 연료로 순환한다.

**[R2] 인기 스타일 학습 및 KPI 추가**
- **표면:** 인기 있는 스타일을 학습하고 KPI를 추가한다.
- **진짜 의도(중요):** 서버·외부 AI가 없으므로 여기서 "학습"은 **머신러닝이 아니라, 클라이언트 로컬에 축적된 행동 신호(즐겨찾기·복사·리액션·재생성)를 통계적으로 집계해 선호 스타일을 도출·반영하는 규칙 기반 개인화**를 의미한다. "KPI 추가"는 v1.0의 KPI 체계를 **고도화 기능에 맞춰 확장**하고, 그 지표를 사용자에게 **대시보드로 가시화**하는 것을 뜻한다.
- **해결 문제:** 개인화 부재 + 성과 가시성 부재.
- **성공한 모습:** 자주 채택되는 공식/톤이 상단 우선노출·배지로 강조되고, 사용자가 자신의 사용 패턴을 지표로 확인한다.
- ⚠️ **기획 유의점:** "AI가 학습한다"는 과장 표현을 지양하고 **"내 사용 데이터 기반 추천"**으로 정직하게 커뮤니케이션한다.

**[R5] 카피 생성 스킬 고도화 (15~20년차 시니어 마케터 관점)**
- **표면:** 생성되는 카피 자체의 품질을 프로 수준으로 끌어올린다.
- **진짜 의도:** v1.0 엔진(단순 템플릿+랜덤)을 **검증된 카피라이팅 프레임워크 + 카피 크래프트(기능→혜택 변환, 구체성/숫자 주입, 파워워드, 감정 트리거, 클리셰 회피) + 품질 스코어링**으로 재설계. 여전히 규칙 기반이되, **시니어 마케터의 사고 절차를 규칙으로 코드화**한다.
- **해결 문제:** 품질 편차·클리셰 반복.
- **성공한 모습:** 첫 생성만으로도 "바로 쓸 만한" 헤드라인 비율이 오르고, 재생성 횟수가 줄어든다.

### 2.2 요구사항 상호관계 (설계 구조도)

```
              ┌───────────────────────────────────────────┐
              │        F14. 카피 엔진 v2 (R5)              │
              │  프레임워크·파워워드·품질스코어 → 생성 품질↑ │
              └───────────────┬───────────────────────────┘
                              │ 생성
                              ▼
   ┌──────────────────────────────────────────────────────┐
   │   F10. 생성 이력 + 입력 메타데이터 (R1 + R4)          │  ← 모든 결과를 맥락과 함께 축적
   └───────────────┬───────────────────────┬──────────────┘
                   │                       │
        (사용자 피드백 부착)          (행동 신호 집계)
                   ▼                       ▼
   ┌────────────────────────┐   ┌───────────────────────────┐
   │ F11. 리액션 & 코멘트    │──▶│ F12. 인기 학습·추천 (R2)    │
   │        (R3)            │   │ F13. KPI 대시보드 (R2)      │
   └────────────────────────┘   └───────────────────────────┘
```

핵심 흐름: **엔진(F14)이 잘 만든다 → 이력(F10)에 맥락과 함께 쌓인다 → 피드백(F11)이 붙는다 → 신호를 학습(F12)해 추천하고, 성과를 지표(F13)로 보여준다.**

### 2.3 핵심 아키텍처 결정 사항 (Design Decisions)

| 결정 | 내용 | 근거 |
|------|------|------|
| **서버리스 유지** | 백엔드·외부 API 도입하지 않음 | v1.0 제약 준수, 진입장벽 최소화 |
| **저장소 승격** | 즐겨찾기(LocalStorage) → **이력/피드백/통계는 IndexedDB** 로 이관 | 이력·코멘트 누적 시 LocalStorage 5MB 한계·동기 I/O 병목 회피 |
| **학습 = 통계 집계** | ML 아님. 로컬 행동 신호의 가중 집계 | 서버·데이터셋 부재, 정직한 UX |
| **데이터 소유권** | 전 데이터 로컬 저장, 외부 전송 없음 | 프라이버시·단일 사용자 도구 특성 |
| **하위 호환** | v1.0 즐겨찾기 데이터를 신규 이력 스키마로 **마이그레이션** | 기존 사용자 데이터 보존 |
| **파일 구조 확장** | `script.js` 단일 → 모듈 논리 분리(엔진/저장소/통계/UI) | 유지보수성 (물리 파일 분리는 선택) |

## 3. 고도화 목표 & 신규 KPI

### 3.1 정성 목표

- **G1.** 좋은 카피를 다시 못 찾는 경험을 0으로 만든다. (이력 100% 보존)
- **G2.** 시스템이 사용자의 스타일 취향을 파악해 **다음 생성을 더 빠르게** 만든다.
- **G3.** 첫 생성 결과의 "즉시 사용 가능" 체감 품질을 끌어올린다.

### 3.2 신규 KPI (R2 "KPI 추가" 직접 대응)

v1.0 KPI(생성 완료율, 만족도, 재사용률 등)에 **고도화 전용 지표**를 추가한다.

**Tier 1 — 이력/재현 지표**
| KPI | 정의 / 측정식 | 목표 |
|-----|--------------|------|
| 이력 재활용률 | (이력에서 재생성·복사·입력불러오기 발생 세션) / (전체 세션) | > 35% |
| 재현 성공률 | "동일 조건 재생성" 실행 횟수 / 이력 조회 횟수 | > 20% |
| 이력 보존율 | 저장된 이력 건수 / 생성 시도 건수 | ≈ 100% |

**Tier 2 — 피드백 지표 (R3 → R2 연결)**
| KPI | 정의 / 측정식 | 목표 |
|-----|--------------|------|
| 리액션 부착률 | 리액션이 달린 카피 수 / 생성 카피 수 | > 25% |
| 코멘트 작성률 | 코멘트가 달린 카피 수 / 생성 카피 수 | > 8% |
| 긍정 리액션 비율 | (👍+❤️+🔥) / 전체 리액션 | 관찰 지표 |

**Tier 3 — 학습/추천 지표 (R2 핵심)**
| KPI | 정의 / 측정식 | 목표 |
|-----|--------------|------|
| 추천 채택률 | 추천(상단노출/배지) 공식이 복사·즐겨찾기·긍정리액션으로 이어진 비율 | > 40% |
| Top 공식 집중도 | 상위 2개 공식이 차지하는 채택 비중 (개인화 강도 관찰) | 관찰 지표 |

**Tier 4 — 엔진 품질 지표 (R5 핵심)**
| KPI | 정의 / 측정식 | 목표 |
|-----|--------------|------|
| 평균 헤드라인 품질점수 | F14 스코어링 엔진 산출 평균 (0~100) | > 75 |
| 첫 생성 채택률 | 재생성 없이 첫 결과를 복사/즐겨찾기한 비율 | > 50% |
| 평균 재생성 횟수 | 세션당 "다시 생성" 클릭 수 (낮을수록 첫 생성 만족↑) | < 1.5 |

### F10. 생성 이력 & 입력 메타데이터 — [Must] (R1 + R4)

**목적:** 모든 생성 결과를 **입력 맥락과 함께** 영속 저장하고, 조회·검색·필터·재실행·삭제할 수 있게 한다.

**사용자 스토리**
- 마케터로서, 과거에 뽑은 카피와 그 입력 조합을 다시 보고 싶다. → 되짚기
- 기획자로서, 좋았던 조합을 그대로 다시 돌리거나 살짝 바꿔 재생성하고 싶다. → 재현·튜닝

**상세 동작**
1. 헤드라인이 생성될 때마다 **이력 항목(GenerationEntry)** 이 자동 저장된다. (별도 저장 버튼 불필요)
2. 각 항목은 아래를 함께 보관한다: 헤드라인 텍스트, **입력 메타(키워드/타깃/혜택/톤)**, 공식(formula), 변형 시드(variationSeed), 품질점수, 생성시각.
3. 이력 패널 UI 제공:
   - **타임라인 리스트** (최신순), 무한 스크롤/페이지네이션
   - **필터:** 공식별 / 톤별 / 즐겨찾기만 / 리액션 있는 것만 / 기간
   - **검색:** 키워드·헤드라인 텍스트 부분일치
   - **항목 액션:** `동일 조건 재생성` · `입력 불러오기(폼에 채우기)` · `복사` · `삭제`
4. **동일 조건 재생성:** 저장된 입력 메타를 그대로 엔진에 재투입하되 새 변형을 산출(R1 재현성의 실사용 지점).

**데이터** → §5 GenerationEntry 스키마 참조

**예외/엣지**
- 이력 상한(예: 500건) 초과 시: 즐겨찾기·리액션 없는 **오래된 항목부터 자동 아카이브/삭제**(사용자 설정 가능), 삭제 전 안내.
- 저장 실패(용량 초과): 토스트 안내 + "이력 정리" 유도.
- v1.0 즐겨찾기 데이터: 최초 실행 시 이력 스키마로 1회 마이그레이션(`isFavorite:true`로 편입).

**수용 기준 (AC)**
- [ ] 생성 시 이력이 100% 자동 저장된다.
- [ ] 이력 항목에서 "동일 조건 재생성"이 원 입력값으로 정확히 동작한다.
- [ ] 공식·톤·즐겨찾기·기간 필터와 텍스트 검색이 동작한다.
- [ ] 새로고침 후에도 이력이 유지된다.

### F11. 리액션 & 코멘트 — [Must] (R3)

**목적:** 이진 즐겨찾기를 넘어, 카피별 **빠른 감정 신호(리액션)** 와 **정성 메모(코멘트)** 를 남긴다. 이 데이터는 F12 학습의 핵심 연료다.

**사용자 스토리**
- 결과가 마음에 들면 한 번 탭으로 표시하고 싶다(리액션).
- "이 카피, 숫자만 넣으면 완벽" 같은 수정 메모를 붙여두고 싶다(코멘트).

**상세 동작**
1. **리액션:** 카드/이력 항목에 이모지 4종 노출 — 👍 좋아요 · ❤️ 최고 · 🔥 강력 · 🤔 애매. 토글 방식, 항목당 1개 선택.
2. **코멘트:** 항목 확장 시 텍스트 입력(최대 200자). 저장/수정/삭제 가능. 작성 시각 표기.
3. 리액션·코멘트 값은 해당 GenerationEntry에 귀속 저장되고, 이력/즐겨찾기에서 함께 표시된다.
4. 리액션은 F12의 **스타일 선호 가중치**에 반영된다. (§7.4 가중치 표)

**예외/엣지**
- 코멘트 XSS 방지: 입력 이스케이프 처리 후 저장/표시.
- 빈 코멘트 저장 차단(공백만 입력 시 무시).
- 200자 초과: 입력 제한 + 카운터 표시.

**수용 기준 (AC)**
- [ ] 리액션 토글이 즉시 반영·저장되고 새로고침 후 유지된다.
- [ ] 코멘트 CRUD가 동작하며 XSS 입력이 무력화된다.
- [ ] 리액션이 인기 학습 가중치에 반영된다.

### F12. 인기 스타일 학습 & 추천 — [Should] (R2)

**목적:** 로컬에 쌓인 행동 신호를 **통계적으로 집계**하여 사용자의 선호 스타일(공식·톤)을 도출하고, 다음 생성에서 이를 **추천·우선노출**한다. (ML 아님 — 규칙 기반 개인화)

**사용자 스토리**
- 내가 자주 채택하는 스타일을 시스템이 알아서 위로 올려주면 좋겠다.

**상세 동작 (학습 로직)**
1. **신호 수집:** 각 공식/톤에 대해 아래 이벤트를 누적 카운트한다 — 복사, 즐겨찾기, 리액션(가중), 재생성(부정 신호), 노출.
2. **선호 점수 산출 (StyleScore):**
   ```
   StyleScore(formula) =
       w_copy   × 복사수
     + w_fav    × 즐겨찾기수
     + w_react  × Σ(리액션 가중치)
     - w_regen  × 재생성수
   ```
   (가중치 기본값은 §7.4)
3. **정규화:** 노출수 대비 정규화하여 신규/소수 노출 공식이 불리하지 않게 보정(신뢰도 하한 적용).
4. **적용:**
   - 결과 영역에서 **선호 상위 공식 카드를 상단 우선 정렬**
   - 상위 공식에 **`👑 인기` 배지** 표시
   - (선택) 생성 시 상위 톤을 기본 선택값으로 프리셋
5. **투명성/제어:** "이 추천은 내 사용 데이터 기반"임을 명시. **추천 끄기 / 학습 데이터 초기화** 제공.

**예외/엣지**
- 콜드 스타트(데이터 부족): 추천 비활성, v1.0 기본 순서 유지. 최소 임계(예: 총 채택 10건) 도달 시 활성화.
- 데이터 편향 안내: 특정 공식에 과집중 시 "다양한 스타일도 시도해보기" 소프트 넛지(선택).

**수용 기준 (AC)**
- [ ] 채택 신호가 StyleScore에 반영되어 상위 공식이 재정렬된다.
- [ ] 콜드 스타트 시 추천이 켜지지 않는다.
- [ ] 추천 끄기·학습 초기화가 동작한다.

### F13. KPI 대시보드 — [Should] (R2)

**목적:** §3.2의 지표를 사용자에게 **가시화**한다. ("KPI 추가" 요구의 사용자 대면 산출물)

**상세 동작**
1. 별도 탭/패널로 제공. 다음을 요약 카드로 표시:
   - 총 생성 수 · 총 채택(복사+즐겨찾기) 수 · 첫 생성 채택률
   - **공식별 채택 분포** (막대)
   - **톤별 선호 분포**
   - 평균 헤드라인 품질점수 추이(간단 스파크라인)
   - 리액션 구성 비율
2. 모두 로컬 데이터 기반 실시간 계산. 외부 분석 도구 불필요.
3. "리포트 내보내기"(선택): 이력+통계를 JSON/CSV로 다운로드(백업 겸용).

**수용 기준 (AC)**
- [ ] 각 지표가 실제 이력/피드백 데이터와 일치한다.
- [ ] 데이터가 없을 때 빈 상태(empty state)를 적절히 안내한다.

### F14. 카피 엔진 v2 — [Must] (R5, 시니어 마케터 관점)

**목적:** 단순 템플릿 치환을 넘어, **검증된 카피라이팅 프레임워크와 크래프트 규칙, 품질 스코어링**을 탑재해 첫 생성 품질을 끌어올린다. 상세 설계는 **§7**에 별도 기술한다.

**핵심 구성 (요약)**
1. 프레임워크 확장: 기존 6공식 위에 PAS·AIDA·BAB·4U·FAB·4C 원리를 규칙화.
2. 크래프트 레이어: 기능→혜택 변환, 구체성·숫자 주입, 파워워드/감정 트리거, 타깃 페르소나 어휘, 클리셰 블랙리스트.
3. 품질 스코어링(0~100): 5축 평가 후 임계 미달 시 자동 재생성.

**수용 기준 (AC)**
- [ ] 모든 생성 헤드라인에 품질점수가 산출된다.
- [ ] 임계(예: 60점) 미달 결과는 노출 전 자동 재생성된다.
- [ ] 클리셰 블랙리스트 문구가 결과에 노출되지 않는다.
- [ ] 동일 입력 반복 생성 시 표현 다양성이 유지된다.

## 7. 카피 엔진 v2 상세 설계 (R5 — 15~20년차 시니어 마케터 관점)

> 시니어 마케터가 헤드라인을 쓸 때의 **사고 절차를 규칙으로 코드화**한다. 여전히 클라이언트·규칙 기반(외부 AI/API 없음)이되, "왜 이 문장이 팔리는가"를 반영한다.

### 7.1 검증된 프레임워크를 공식에 매핑

| 기존 공식 | 결합 프레임워크 | 시니어 관점의 작성 원리 |
|-----------|----------------|------------------------|
| 숫자형 | **4U** (Ultra-specific) | 막연함을 죽이고 정량·구체로. "많이" → "3배", "빠르게" → "5분 안에" |
| 질문형 | **AIDA** (Attention) | 독자의 현재 고민을 되묻는 질문으로 주의 포획. Yes를 유도하는 질문 설계 |
| 혜택형 | **FAB / BAB** | 기능(Feature)이 아니라 **결과·감정(Benefit)** 과 Before→After 대비 |
| 공포소구형 | **PAS** (Problem-Agitate-Solution) | 문제를 짚고(과하지 않게) 손실 회피 심리를 자극한 뒤 해결 제시 |
| 호기심형 | **Curiosity Gap** | 정보 공백을 만들되 낚시성/과장은 금지(신뢰 훼손 방지) |
| 트렌드형 | **4C** (Credible) | 시의성 + 근거로 "지금·여기" 당위 부여 |

### 7.2 카피 크래프트 규칙 (생성 파이프라인)

생성 시 아래 순서로 문장을 정련한다.

1. **기능 → 혜택 변환:** 입력 혜택을 "사용자의 하루에 생기는 변화"로 치환.
   - 예) `혜택=배터리` → "충전 걱정 없이 하루 종일"
2. **구체성·숫자 주입:** 숫자/기간/비율 슬롯을 우선 채워 추상어를 대체.
3. **파워워드 & 감정 트리거 적용:** 상황에 맞는 어휘 삽입(§7.3).
4. **타깃 페르소나 어휘:** `target` 값에 따라 어조·호칭·관심사 반영(예: "직장인" → 출퇴근·업무 효율 프레임).
5. **클리셰 블랙리스트 필터:** 진부한 상투구를 차단·치환.
6. **길이·가독성 정련:** 모바일 가독 기준(권장 ≤ 30자 핵심부) 초과 시 압축.

### 7.3 파워워드 / 감정 트리거 사전 (예시)

| 카테고리 | 어휘 예시 |
|----------|-----------|
| 신뢰 | 검증된, 보장, 이미 N만 명, 실제 후기 |
| 긴급/희소 | 지금, 마감, 단 하루, 놓치면 |
| 이득 | 무료, 절약, 2배, 되찾다 |
| 호기심 | 비밀, 아무도 몰랐던, 진짜 이유, 반전 |
| 손실회피 | 실수, 아직도, 손해, 뒤처지다 |

### 7.4 신호 가중치 (F12 학습 연동 기본값)

| 신호 | 기호 | 기본 가중치 | 의미 |
|------|------|:---:|------|
| 복사 | w_copy | +3 | 실사용 강신호 |
| 즐겨찾기 | w_fav | +2 | 보관 의향 |
| 리액션 🔥/❤️ | w_react | +2 | 강한 긍정 |
| 리액션 👍 | w_react | +1 | 약한 긍정 |
| 리액션 🤔 | w_react | -1 | 애매/약부정 |
| 재생성 | w_regen | -1 | 첫 결과 불만족 |

### 7.5 헤드라인 품질 스코어링 (0~100)

5개 축을 가중 합산해 점수화하고, **임계(기본 60점) 미만이면 노출 전 자동 재생성**한다.

| 평가 축 | 배점 | 판정 규칙(휴리스틱) |
|---------|:---:|------|
| 명확성 (Clear) | 25 | 핵심 메시지 1개, 모호어 없음 |
| 구체성 (Specific) | 20 | 숫자/고유명/기간 포함 여부 |
| 혜택성 (Benefit-led) | 25 | 기능 나열이 아닌 결과·감정 중심 |
| 긴급/매력 (Compelling) | 15 | 파워워드·트리거 포함 |
| 차별성 (Distinct) | 15 | 클리셰 블랙리스트 미포함, 표현 신선도 |

### 7.6 클리셰 블랙리스트 (예시)

`"당신의 인생을 바꿀"`, `"상상 그 이상"`, `"최고의 선택"`, `"놓치면 후회"`(과다 반복 시) 등 → 감점·치환 대상으로 관리하고 주기적으로 갱신한다.

## 8. 우선순위 & 릴리스 계획

## 9. 예외 · 엣지 케이스 & 제약

- **저장 용량 초과:** 이력 자동 아카이브 정책 + 사용자 정리 유도. (즐겨찾기·리액션 항목은 보호 우선)
- **다중 탭 동시 사용:** IndexedDB 쓰기 충돌 방지(마지막 쓰기 우선 + 갱신 이벤트 동기화).
- **브라우저 시크릿 모드/저장소 차단:** 영속화 불가 안내, 세션 한정 동작으로 폴백.
- **콜드 스타트:** 추천/대시보드 빈 상태 처리.
- **개인정보:** 전 데이터 로컬 보관·외부 미전송 명시. 초기화 수단 제공.
- **하위호환:** v1.0 즐겨찾기 100% 무손실 마이그레이션.
